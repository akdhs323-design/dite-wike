Overview

디테봇(Dite Bot) 개요

디테봇은 디스코드(Discord) 생태계에서 활동하는 고성능 하이브리드 대화 AI 시스템입니다. Gemini-3.1-Flash-Lite 모델을 메인 신경망으로 활용하며, 디스코드 봇 구동 중 발생할 수 있는 자원 및 계정 한도 초과 오류를 능동적으로 우회하기 위해 다중 API 로테이션 및 실시간 검증 모듈을 내장하고 있습니다. 단순 응답을 넘어 로컬 SQLite 데이터베이스를 기반으로 동작하여 유저의 행동 양식, 누적 채팅, 제재 상태 및 영속 기억들을 실시간 보정 동기화합니다.

다중 API 로테이션

API 키를 풀(Pool) 단위로 유지하고, 순차 로테이션을 가동하여 특정 키 만료 시 401 오류를 탐지해 실시간 격리합니다.

하이브리드 병렬 검색

DuckDuckGo Lite GET 및 Wikipedia API를 비동기 병렬 조회하여 최적화된 출처 각주 마크다운 리포트를 자동 완성합니다.

비파괴 소프트 삭제

사용자나 관리자가 기억 초기화를 시행할 때, 실제 런타임 풀에서만 삭제하고 DB 아카이브 격리실에 영구 백업 보존합니다.

스마트 타임 정지 가드

`30m`, `5h`, `7d` 등의 한글/영문 융합 정지 기한 포맷을 파싱하여 타겟 사용자 ID를 정교하게 차단 관리합니다.

Core Architecture

디테 아키텍처 상세 백서

다중 키 검증 및 동적 폴트 톨러런스 (Fault Tolerance)

디테봇은 Google Gemini API의 호출량 상한(Rate Limit) 및 폐기 계정으로 인한 작동 중단을 예방하기 위해 독창적인 키 순환 메커니즘을 사용합니다.

# ver_7.3.py 다중 키 라운드로빈 로테이션 & 격리 슈도 코드

1. on_ready 단계에서 check_and_validate_api_keys() 비동기 백그라운드 태스크 실행

2. 등록된 모든 키에 가벼운 "ping" 테스트 호출을 수행하여 유효성 선 검사 시행

3. 만약 API 응답 오류 중 "401" / "unauthenticated" / "service account is deleted" 감지 시 즉각 해당 키 차단

4. 런타임 클라이언트 리스트(active_keys)에서 제거 격리 후 설정 창에 경고 기록

5. 대화 처리 시 key_index = (key_index + 1) % len(active_keys) 기법으로 라운드로빈 자동 우회 순환

초고속 병렬 웹 정보 합성 파이프라인 (Lite GET 기반)

공신력 있는 실시간 데이터를 수집하기 위해, 복잡한 셀레늄(Selenium) 동적 브라우저 구동보다 수십 배 빠른 **DuckDuckGo Lite GET 및 Wikipedia API**를 연동합니다.

// 1단계: 검색어 융합 & 우회

사용자가 질문한 문맥을 분석하여 AI가 직접 최적의 검색어 추출 (나무위키, 위키백과 등 검색 지점 주입 강제화)

특정 단어(삼성, 디테, Notion 개인정보 등) 감지 시 불필요한 네트워크 대기 없이 하드코딩 메모리 캐시 데이터를 자동 즉시 매핑하여 속도 보장

// 2단계: 자가 치유(Self-Healing) 복원

DuckDuckGo가 차단되거나 타임아웃 발생 시, 시스템 다운을 배제하고 즉각 generate_fallback_search_data() 비동기 백업 엔진 작동

Gemini 내장 지식을 사용해 실제와 거의 100% 흡사한 지식 스니펫 및 나무위키 주소를 합성하여 정보 흐름 보전

영속 기억 보존실 (Real-time DB Syncer)

디테봇은 대화 기록이 유실되는 현상을 막기 위해 사용자별 런타임 기억을 데이터베이스와 실시간 영속 동기화합니다.

  • 메모리 제한 및 Capping: API 토큰 절약을 위해 유저별 세션 대화는 최대 50건(MAX_HISTORY)으로 제한 관리됩니다. 초과 시 선입선출(FIFO) 방식으로 앞 대화를 소멸시킵니다.
  • 실시간 백업 트리거: 유저와 메시지를 주고받은 즉시, save_user_memory_to_db()를 비동기로 호출하여 SQLite `user_active_memories` 테이블에 직렬화된 JSON으로 영구 저장합니다.
  • 비파괴 격리 삭제 기법: 유저가 초기화를 요청할 경우 런타임 풀에서는 삭제되지만, 데이터베이스 `deleted_memories` 아카이브로 온전하게 백업 보존되어 추후에 관리자 패널에서 복구 및 트레이싱 조회가 가능합니다.

KST 자정 리셋 스케줄러 & 로그 다이어트 (DB Diet)

봇의 데이터베이스가 끝없이 팽창하여 속도가 저하되는 문제점을 해결하기 위해 KST 자정(오전 12:00) 주기적 클린업 프로세스가 자동으로 실행됩니다.

일일 활동 랭킹 초기화

매일 자정 KST 날짜가 변경되면 `chat_counts` 테이블의 누적 랭킹 정보 데이터를 안전하게 비워 오늘 하루의 새 활동 랭킹을 시작합니다.

30일 로그 청소 (DB 다이어트)

데이터가 장기 적재되어 쓰기 성능이 저하되는 상황을 막고자, 등록 시간 기준 30일이 초과한 정밀 데이터 로그(`message_logs`)는 자동 청소 및 소멸 처리됩니다.

Quick Deployment

직접 배포 및 가동 안내

디테봇은 간결하게 가상 서버(VPS), 라즈베리 파이, 토이 서버 등 환경에서 즉시 구동될 수 있도록 개발되었습니다. 설치를 완료한 후 아래 실행 커맨드를 이용해 패키지를 구축하세요.

가동 절차

  1. Python 3.11 이상 버전 및 SQLite 패키지가 활성화되어 있는지 확인합니다.
  2. pip install -r requirements.txt 명령어를 실행하여 외부 종속성을 수집합니다.
  3. 같은 루트 경로에 디스코드 봇 토큰 및 제반 환경이 기입된 .env 파일을 설정합니다.
  4. python ver_7.3.py 명령어로 즉시 가동을 시작합니다.
bash 실행 커맨드 
# 1. 소스 디렉터리로 이동
cd /your/ditebot/directory

# 2. 필수 라이브러리 일괄 자동 다운로드
pip install discord.py google-genai python-dotenv beautifulsoup4 aiohttp selenium

# 3. 디테봇 v7.3 서버 데몬 백그라운드 구동 (로그 보전)
nohup python ver_7.3.py > ditebot.log 2>&1 &
