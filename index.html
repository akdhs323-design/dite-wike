Overview

디테봇(Dite Bot) 개요

디테봇은 디스코드(Discord) 생태계에서 활동하는 청소년 맞춤형 고성능 하이브리드 대화 AI 시스템입니다. Gemini-3.1-Flash-Lite 모델을 메인 신경망으로 가동하며, 디스코드 봇 구동 중 발생할 수 있는 자원 및 계정 한도 초과 오류를 능동적으로 우회하기 위해 다중 API 로테이션 및 실시간 검증 모듈을 내장하고 있습니다. 단순 응답을 넘어 로컬 SQLite 데이터베이스를 기반으로 동작하여 유저의 행동 양식, 누적 채팅, 제재 상태 및 영속 기억들을 실시간 보정 동기화합니다.

API 로테이션

API 키를 풀(Pool) 단위로 유지하고, 순차 로테이션을 가동하여 특정 키 만료 시 401 오류를 탐지해 실시간 격리합니다.

하이브리드 병렬 검색

DuckDuckGo Lite GET 및 Wikipedia API를 비동기 병렬 조회하여 최적화된 출처 각주 마크다운 리포트를 자동 완성합니다.

실시간 1:1 DM 티켓

이용정지 유저가 봇에게 DM을 전송하면 실시간 소명 티켓방이 개설되며, 관리진 개인 메신저와 1:1 양방향 릴레이 연동이 지원됩니다.

신규 가이드 & 온점 가드

봇을 처음 쓰는 신규 유저에게 웰컴 임베드를 즉각 자동 표기하며, 명령어 충돌 방지용 온점(`.`) 차단 처리가 탑재되었습니다.

Core Architecture

디테 아키텍처 상세 백서 (v9.0)

다중 키 검증 및 동적 폴트 톨러런스 (Fault Tolerance)

디테봇은 Google Gemini API의 호출량 상한(Rate Limit) 및 폐기 계정으로 인한 작동 중단을 예방하기 위해 독창적인 키 순환 메커니즘을 사용합니다.

// ver_9.0.py 다중 키 라운드로빈 로테이션 & 격리 슈도 코드

1. on_ready 단계에서 check_and_validate_api_keys() 비동기 백그라운드 태스크 실행

2. 등록된 모든 키에 가벼운 "ping" 테스트 호출을 수행하여 유효성 선 검사 시행

3. 만약 API 응답 오류 중 "401" / "unauthenticated" / "service account is deleted" 감지 시 즉각 해당 키 차단

4. 런타임 클라이언트 리스트(active_keys)에서 제거 격리 후 설정 창에 경고 기록

5. 대화 처리 시 key_index = (key_index + 1) % len(active_keys) 기법으로 라운드로빈 자동 우회 순환

초고속 병렬 웹 정보 합성 파이프라인 (Lite GET 기반)

공신력 있는 실시간 데이터를 수집하기 위해, 복잡한 셀레늄(Selenium) 동적 브라우저 구동보다 수십 배 빠른 **DuckDuckGo Lite GET 및 Wikipedia API**를 연동합니다.

1단계: 검색어 융합 & 우회

사용자가 질문한 문맥을 분석하여 AI가 직접 최적의 검색어 추출 (나무위키, 위키백과 등 검색 지점 주입 강제화)

삼성, 디테, Notion 정책 등 고정 트리거 감지 시 네트워크 대기 없이 하드코딩 메모리 캐시 데이터를 즉시 매핑하여 속도 극대화

2단계: 자가 치유(Self-Healing) 복원

DuckDuckGo가 차단되거나 타임아웃 발생 시 시스템 다운을 배제하고 즉각 generate_fallback_search_data() 비동기 백업 엔진 가동

Gemini 내장 지식을 활용해 실제와 거의 일치하는 지식 스니펫 및 나무위키 주소를 합성하여 정보 중단 방지

영속 기억 보존 및 신규 유저 판별 가드

디테봇은 대화 기록 유실 방지와 첫 사용자 안내를 위한 스마트 가이드를 동시에 탑재하고 있습니다.

  • 실시간 백업 트리거: 유저와 메시지를 주고받은 즉시, save_user_memory_to_db()를 비동기로 호출하여 SQLite `user_active_memories` 테이블에 직렬화된 JSON으로 영구 저장합니다.
  • 신규 유저 감지 모듈 (`is_new_user`): 사용자가 보낸 대화 역사 및 삭제 아카이브를 전면 스캔하여 기록이 전혀 존재하지 않는 유저에게만 첫 질문 시 가이드라인 및 공식 위키 주소가 담긴 웰컴 카드를 동적으로 우선 송신합니다.

DM 실시간 티켓 소통 및 양방향 메신저 릴레이

정지 조치된 악성 의심 유저의 소명 기회를 위해 고도로 지능화된 1:1 실시간 양방향 메신저 연동 시스템을 구비하고 있습니다.

유저 DM ➔ 관리진 실시간 배송

정지 유저가 디테봇 DM에 소명 문구를 전송하면, SQL에 티켓을 등록한 후 등록된 관리자들의 개인 DM창으로 임베드 카드를 즉각 푸시 전송합니다.

관리자 답장 ➔ 유저 실시간 DM 역송

관리자가 전송받은 제어 버튼 `[💬 답장하기]`를 클릭해 내용을 입력하면, 봇이 대상 유저의 개인 DM으로 정교한 보안 피드백 블록을 즉시 전송합니다.

User & operator manual

디테봇 완전 가이드 & 운용 매뉴얼

일반 유저의 올바른 봇 활용 방법부터 봇 운영팀이 이의제기 및 설정 패널을 제어하기 위한 구체적인 단계별 액션 가이드라인입니다.

1. **봇 호출법**: 전용 채널 이외의 서버 공용 채널에서는 디테야 혹은 디테라고 이름을 먼저 부르고 질문해야 똑똑하게 반응합니다.

2. **대화 초기화**: 대화 기억을 완전히 새롭게 전환하고 싶다면 슬래시 명령어 /기억초기화 또는 일반 채팅창에 !기억초기화를 전송하여 대화를 포맷하세요.

3. **주의 사항**: 봇 명령어 전송 시 다른 봇 제어 간섭을 방지하기 위해 문장 앞에 온점(.)을 기입하여 말을 걸면 디테봇은 절대 대답하지 않고 그냥 통과시킵니다.

1. **설정 패널 호출**: 슬래시 명령어 /설정은 오직 developer_ids에 등록된 관리자만 소환하고 변경 사항을 저장할 수 있습니다. 비인가자가 입력 시 전격 차단 경고가 반환됩니다.

2. **이의제기 승인 및 해제**: 유저가 제출한 소명서를 대시보드 탭에서 심사 시, 수락 시 즉시 차단이 전면 해제되며 상대 유저의 개인 DM으로 "승인 해제" 축하 안내가 자동 송수신됩니다.

3. **1:1 양방향 티켓 대화**: 반려나 즉시 해제가 애매한 상황일 때 [💬 답장하기] 버튼을 이용해 조건부 참작 메시지나 경고를 전송해 유저와 계속 대화를 유지할 수 있습니다.
